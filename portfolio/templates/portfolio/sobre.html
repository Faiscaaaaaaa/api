Sobre esta Aplicação

Nesta página apresento a aplicação de forma completa: a modelação no papel, a arquitetura Django, a estrutura do site, as tecnologias usadas, o repositório GitHub, a solução em produção com Neon e Cloudinary e a pipeline de CI/CD.

O site também disponibiliza uma página dedicada de videotutoriais com demonstrações do desenvolvimento, arquitetura e uso de Codespaces.

1. Visão Geral da Aplicação

Este portfólio foi construído como uma aplicação Django com três apps principais:

  • portfolio: páginas públicas de apresentação, making-of, sobre, cursos e dados pessoais.
  • artigos: funcionalidades de blog, artigos, comentários, likes e ratings médios.
  • accounts: autenticação, registo de utilizadores e gestão do grupo bloggers.

A aplicação suporta autenticação, autorização por grupos, CRUD de conteúdos e exibição responsiva, reunindo front-end e back-end numa solução integrada.

2. Modelação no Papel e Arquitetura Django

Diagrama MVT:

A arquitetura usa o padrão MVT, onde as Models representam os dados, as Views tratam pedidos e lógica e os Templates geram o HTML para a interface do utilizador.

Esta separação facilita manutenção, permite reutilizar componentes e ajuda a desenvolver funcionalidades como a lista de artigos, o formulário de registo e a página de making-of de forma modular.

Diagrama Entidade-Relacionamento:

Componentes principais: utilizadores, artigos, comentários, ratings, projetos, licenciaturas, unidades curriculares, tecnologias e making-of.

3. Tecnologias Utilizadas

Django 4.2

Framework principal do backend, responsável pelo ORM, autenticação, views e templates.

Python

Linguagem usada para lógica, modelos e manipulação de dados.

HTML/CSS/JavaScript

Construção das páginas, navegação, menus dropdown e responsividade.

GitHub

Versão e histórico do código, uso de branches e commits.

Neon PostgreSQL

Base de dados em produção para dados de utilizadores, artigos e interações.

Cloudinary

Armazenamento externo de media e imagens para não sobrecarregar o servidor de aplicação.

4. Estrutura do Site

O site é organizado em páginas principais que representam os domínios desta aplicação:

  • Landing: página inicial de apresentação com navegação rápida.
  • Pessoal: competências, formações, interesses e contactos.
  • Making Of: registos de desenvolvimento e reflexão crítica.
  • Sobre: esta página técnica com arquitetura e tecnologias.
  • Cursos: licenciaturas, unidades curriculares e projetos.
  • Artigos: blog com autor, data, comentários e ratings.
  • Login / Registo: autenticação e grupo de bloggers para criação de artigos.

5. Arquitetura de Produção

Em produção, o app Django corre de forma isolada, com a base de dados em Neon PostgreSQL e as imagens armazenadas em Cloudinary. Esta separação garante maior performance e fiabilidade.

O backend lida com autenticação, artigos, comentários e conteúdos estáticos, enquanto Neon guarda dados relacionais e Cloudinary gere media de forma escalável.

6. Pipeline CI/CD

O fluxo de entrega segue um processo automatizado e controlado:

  • Desenvolvimento local: o código é escrito e testado em Codespaces ou localmente.
  • Repositório GitHub: o código é versionado e guardado no GitHub.
  • GitHub Actions: executa validações, testes e build automático.
  • Deploy: o código aprovado é publicado em produção, mantendo a aplicação atualizada.

7. Aprendizagens e Desafios

Dificuldades superadas

Resolvi problemas como a necessidade de Pillow para ImageField, a correção do uso de `markdownify` e a configuração do grupo `bloggers`.

Motivos de orgulho

Fico orgulhoso do blog com comentários e ratings, do menu responsivo e da arquitetura clara de produção.

Uso de IA Generativa

Usei IA para apoiar a escrita do relatório, estruturar o conteúdo da página e melhorar a explicação da arquitetura.
