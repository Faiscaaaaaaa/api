Arquitetura e pipeline CI/CD

Abaixo está um desenho inspirado no esquema de papel que mostra como o sistema está organizado: desenvolvimento local, repositório de código, pipeline de CI/CD, produção e armazenamento de dados/media.

1. Browser / Frontend

É a interface onde o utilizador interage com o portfolio. O HTML, CSS e JavaScript são servidos pelo Django e renderizados no navegador.

2. Codespaces

O ambiente de desenvolvimento onde o código foi escrito, testado e versionado antes do push para o repositório.

3. GitHub Repository

Armazena todo o código, historial de versões e as branches usadas durante o desenvolvimento.

4. GitHub Actions

Automatiza testes, build e deploy. Esta pipeline garante que a publicação acontece de forma repetível e controlada.

5. Produção em Docker

O app Django corre dentro de um container Docker que entrega a aplicação e os ficheiros estáticos em produção.

6. Neon PostgreSQL

Guarda os dados essenciais do projeto: utilizadores, artigos, comentários, ratings e configurações.

7. Cloudinary

Armazena imagens e media fora do servidor principal, garantindo carregamento rápido e gestão de ficheiros mais simples.

Este desenho e a descrição ajudam a mostrar claramente o fluxo completo: desenvolvimento local em Codespaces, push para GitHub, execução de CI/CD, deploy em Docker, e consumo do site pelo browser com dados armazenados em Neon e media em Cloudinary.
